# Chapter 6 — Use Cases

> Real stories from real practitioners. How they use OpenClaw, what they configured, what happened.

## How to submit your use case

Use the [Use Case template](https://github.com/alexwill87/openclawfieldplaybook/issues/new?template=suggestion.yml) — label your Issue `use-case`.

**Format:**
```
Context:     Who you are, what your business does
Problem:     What you needed to automate or solve
Setup:       What you configured (system prompt, tools, triggers)
Result:      What changed concretely
Template:    The prompt or config block others can reuse
```

## Use cases we are looking for

These are the most requested — if you have experience with any, your contribution will be especially valuable:

- **Freelance consultant** — client communication, proposal drafting, time tracking
- **E-commerce seller** — order management, customer support, inventory monitoring
- **Developer** — code review assistant, documentation agent, PR summariser
- **Enterprise team (French-speaking)** — internal knowledge base, meeting notes, HR workflows
- **Artisan / craftsperson** — client follow-up, quote generation, supplier management
- **Startup founder** — investor updates, competitive monitoring, hiring pipeline
