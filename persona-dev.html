OpenClaw pour les développeurs

Vous savez ce qu'est Docker. Allons droit au but.

Ce playbook est votre runbook. 92 sections, des commandes copiables, des décisions documentées. Pas de blabla marketing, pas de slides, pas de "vision". Juste ce qu'il faut pour installer, configurer et faire tourner un agent OpenClaw sur un VPS.

Ce que vous allez installer

ComposantRole
OpenClaw + OpenRouterFramework d'agent + accès multi-modèles (Claude, GPT, Mistral, Llama)
HashiCorp VaultGestion des secrets (clés API, tokens, credentials)
PostgreSQLSource de vérité (mémoire, configuration, logs)
TailscaleVPN mesh (zéro exposition publique)
systemdGateway OpenClaw (service système)
GitVersionning de la configuration agent

Le tout sur un VPS Ubuntu 24.04.

En combien de temps ?

PhaseDurée estimée
Installation (chapitre 2, 19 sections)2 à 3 heures
Configuration (chapitre 3)1 à 2 heures
Personnalisation (chapitre 4)Continu

Le chapitre 2 a 19 sections. Vous pouvez les suivre dans l'ordre ou sauter celles que vous maîtrisez déjà.

Si vous avez déjà un VPS

Le playbook gère les configurations existantes :

  • Docker installé ? Le playbook le détecte et passe à la suite.
  • Tailscale connecté ? Parfait, une étape de moins.
  • Node.js présent ? La section 2.15 détecte automatiquement nvm ou node global et s'adapte.
  • Ports occupés ? Les checks de port évitent les conflits avec vos services existants.
