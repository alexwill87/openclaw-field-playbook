Declaration d'accessibilite

Le projet OpenClaw Field Playbook s'engage a rendre son contenu accessible au plus grand nombre, y compris les personnes en situation de handicap visuel, moteur ou cognitif.

Cette page documente les normes suivies, les mesures en place, et comment signaler un probleme ou proposer une amelioration.

Normes et referentiels de reference

Les corrections d'accessibilite appliquees a ce site s'appuient sur trois referentiels reconnus a l'echelle mondiale :

Referentiel Organisation Description Lien
WCAG 2.1 W3C / WAI Web Content Accessibility Guidelines -- la reference mondiale. Trois niveaux : A, AA, AAA. Le niveau AA est le standard vise par la plupart des legislations. w3.org/TR/WCAG21
WAI-ARIA 1.2 W3C / WAI Accessible Rich Internet Applications -- specification technique pour rendre les composants interactifs (menus, onglets, accordeons) accessibles aux technologies d'assistance. w3.org/TR/wai-aria-1.2
RGAA 4.1 DINUM (France) Referentiel General d'Amelioration de l'Accessibilite -- declinaison operationnelle francaise des WCAG, obligatoire pour les services publics. Utilise comme reference complementaire pour le contexte francophone. accessibilite.numerique.gouv.fr
ARIA Authoring Practices W3C / WAI Patterns de conception recommandes pour les composants interactifs courants (accordeons, navigation, recherche, etc.). w3.org/WAI/ARIA/apg
Section 508 U.S. Access Board Standard americain d'accessibilite numerique, aligne sur les WCAG 2.0 niveau AA. Reference pour les projets a audience internationale. section508.gov
EN 301 549 ETSI (Europe) Norme europeenne harmonisee pour l'accessibilite des TIC, base juridique de la directive europeenne sur l'accessibilite du web. etsi.org/EN 301 549

Ce qui est en place

Les mesures suivantes ont ete implementees sur l'ensemble du site :

Structure et navigation

  • Lien "Aller au contenu principal" (skip-link) -- permet de passer directement au contenu sans traverser la navigation. Visible uniquement au focus clavier. (WCAG 2.4.1 -- Niveau A)
  • Landmarks semantiques -- <header>, <nav>, <main>, <aside>, <footer> avec les roles ARIA correspondants (banner, navigation, contentinfo). (WCAG 1.3.1 -- Niveau A)
  • Hierarchie des titres -- aucun saut de niveau (h1 → h2 → h3), permettant une navigation structuree par lecteur d'ecran. (WCAG 1.3.1 -- Niveau A)
  • Fil d'Ariane avec aria-label="Fil d'Ariane". (WCAG 2.4.8 -- Niveau AAA)
  • Attribut lang="fr" sur toutes les pages francophones, lang="en" sur les pages anglophones. (WCAG 3.1.1 -- Niveau A)

Composants interactifs

  • Sections depliables -- utilisent des <button> natifs avec aria-expanded mis a jour dynamiquement, au lieu de <div onclick>. (WCAG 4.1.2 -- Niveau A, ARIA Authoring Practices : Accordion)
  • Toggle de langue -- converti de <a href="javascript:void(0)"> en <button> natif avec aria-label. (WCAG 4.1.2 -- Niveau A)
  • Menu mobile -- aria-expanded mis a jour dynamiquement a l'ouverture/fermeture. (WCAG 4.1.2 -- Niveau A)
  • Champ de recherche -- associe a un <label> (visuellement masque via .sr-only) et a un aria-label, dans un conteneur role="search". (WCAG 1.3.1, 4.1.2 -- Niveau A)
  • Boutons "Copier" et "Retour en haut" -- pourvus de aria-label descriptifs. (WCAG 4.1.2 -- Niveau A)

Clavier et focus

  • Focus visible -- tous les elements interactifs affichent un contour bleu (outline: 2px solid) au focus clavier via :focus-visible. (WCAG 2.4.7 -- Niveau AA)
  • Pas de piege clavier -- aucun composant ne bloque la navigation au clavier. (WCAG 2.1.2 -- Niveau A)

Contenu

  • Contraste de couleurs -- texte principal #1a1a1a sur fond #ffffff (ratio 16.7:1, depasse le niveau AAA). Mode sombre : #e5e5e5 sur #0a0a0a (ratio 18.1:1). (WCAG 1.4.3 -- Niveau AA)
  • Liens descriptifs -- aucun lien generique ("cliquez ici"). Tous les liens decrivent leur destination. (WCAG 2.4.4 -- Niveau A)
  • Tableaux accessibles -- utilisation de <thead>, <th> pour les en-tetes. (WCAG 1.3.1 -- Niveau A)
  • Liens externes -- attribut rel="noopener noreferrer" sur tous les liens target="_blank". Indication visuelle (fleche) et textuelle (texte masque "ouvre un nouvel onglet" pour les lecteurs d'ecran). (WCAG 3.2.5 -- Niveau AAA)
  • SVG decoratifs -- masques avec aria-hidden="true" et focusable="false" pour eviter les annonces parasites. (WCAG 1.1.1 -- Niveau A)

Recherche accessible

  • Navigation clavier dans les resultats : fleches haut/bas pour parcourir, Entree pour selectionner, Echap pour fermer. (WCAG 2.1.1 -- Niveau A)
  • Annonce vocale du nombre de resultats via aria-live="polite". (WCAG 4.1.3 -- Niveau AA)
  • Combobox ARIA avec role="combobox", aria-expanded, aria-activedescendant, et role="listbox" sur les resultats. (ARIA Authoring Practices : Combobox)

Annonces dynamiques (aria-live)

  • Copie de code -- "Code copie dans le presse-papier" annonce via aria-live="polite". (WCAG 4.1.3 -- Niveau AA)
  • Changement de theme -- "Mode sombre active" / "Mode clair active" annonce vocalement. (WCAG 4.1.3 -- Niveau AA)
  • Resultats de recherche -- nombre de resultats annonce a chaque saisie. (WCAG 4.1.3 -- Niveau AA)

Menu mobile

  • Piege de focus -- quand le menu est ouvert, le focus reste a l'interieur (Tab boucle). (WCAG 2.1.2 -- Niveau A)
  • Touche Echap -- ferme le menu et remet le focus sur le bouton d'ouverture. (WCAG 2.1.1 -- Niveau A)
  • Focus automatique -- le premier lien du menu recoit le focus a l'ouverture.

Support du mode sombre

  • Detection automatique via prefers-color-scheme: dark
  • Bascule manuelle via bouton avec aria-label contextuel ("Passer en mode clair" / "Passer en mode sombre")
  • Annonce vocale du changement de theme
  • Preference sauvegardee en localStorage (pas de cookie)

Pages multilingues

  • Les pages anglaises (/en/) utilisent des labels et skip-links en anglais ("Skip to main content", "Search the playbook", etc.)
  • Les pages francoises utilisent des labels en francais
  • L'attribut lang est correct sur chaque page

Niveau de conformite vise

Ce site vise la conformite WCAG 2.1 niveau AA. Certains criteres de niveau AAA sont egalement respectes (fil d'Ariane, contrastes eleves).

Le site n'a pas encore fait l'objet d'un audit formel par un tiers. L'evaluation actuelle est basee sur :

  • Revue manuelle du code source
  • Tests de navigation au clavier
  • Verification de la structure avec les outils de developpement des navigateurs

Limites connues

Les points suivants sont identifies mais pas encore resolus :

  • Tests avec lecteurs d'ecran reels -- le site n'a pas encore ete teste avec NVDA, JAWS, ou VoiceOver. Des retours d'utilisateurs sont bienvenus.
  • Audit automatise -- pas encore de passage systematique avec des outils comme axe-core, Lighthouse Accessibility, ou WAVE.
  • Captions sur les tableaux -- les tableaux ont des <thead> et <th> corrects mais n'ont pas encore d'element <caption> pour decrire leur contenu.
  • Iframe Giscus -- le widget de commentaires (tiers) n'a pas de title sur son iframe. Son accessibilite depend de l'implementation de Giscus.

Signaler un probleme d'accessibilite

Si vous rencontrez une difficulte d'acces ou d'utilisation sur ce site :

  1. Ouvrez une issue sur GitHub avec le label accessibility : Signaler un probleme d'accessibilite
  2. Decrivez le probleme, le navigateur et la technologie d'assistance utilises
  3. Si possible, indiquez le critere WCAG concerne

Toute contribution est bienvenue -- meme un simple retour d'experience avec un lecteur d'ecran nous aide a progresser.
