Chapitre 5 -- Maintenance

Garder votre setup OpenClaw performant, fiable et digne de confiance dans la duree.

Partie A -- Operations quotidiennes

  • 5.1 -- Health check quotidien Script diagnostic. Automatiser avec cron. Alertes Telegram.

  • 5.2 -- Gestion des logs Ou les trouver, comment les lire, rotation, quoi surveiller.

  • 5.3 -- Backups pg_dump, snapshots Hetzner, fichiers critiques. Test de restauration.

Partie B -- Maintenance agent

  • 5.4 -- Revoir le system prompt Frequence, declencheurs, quoi preserver vs reecrire.

  • 5.5 -- Derive de la memoire Memoire obsolete ou contradictoire. Nettoyage periodique.

  • 5.6 -- Quand l'agent se trompe Protocole : diagnostic, correction, prevention.

  • 5.7 -- Mettre a jour les integrations API qui change. Adapter sans casser.

Partie C -- Maintenance infra

  • 5.8 -- Mises a jour systeme Ubuntu, Docker, Node.js, Vault. Quand NE PAS mettre a jour.

  • 5.9 -- Rotation des secrets Pourquoi, comment, frequence.

  • 5.10 -- Monitoring et alertes Simple (cron+Telegram), intermediaire (Uptime Kuma), avance (Grafana).

  • 5.11 -- En cas de panne Tableau diagnostic par symptome. Pour chaque : diagnostic, fix, prevention.

Partie D -- Evolution

  • 5.12 -- Quand ajouter un deuxieme agent Signes, structure, isolation, communication.

  • 5.13 -- Migrer vers un autre modele Comment tester sans casser. Comparaison A/B. Garder le fallback.

  • 5.14 -- Mesurer le ROI Temps gagne, decisions ameliorees, erreurs evitees.

Specifications locales : voir sections/07-localisation/
