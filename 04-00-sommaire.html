Chapitre 4 -- Personnalisation

Faire de votre agent LE votre -- votre voix, vos regles, vos workflows, votre identite.

Partie A -- Identite et voix

  • 4.1 -- Ecrire votre system prompt Le texte le plus important que vous ecrirez pour votre agent.

  • 4.2 -- Personnalite et ton Comment le faire parler comme vous -- et quand ne pas le faire.

  • 4.3 -- Iteration : la premiere version ne sera pas la bonne Normal. Comment converger en 2-3 rounds vers un prompt qui sonne juste.

Partie B -- Systeme de taches

  • 4.4 -- Pourquoi un systeme de taches Agent sans taches = conseiller. Agent avec taches = partenaire.

  • 4.5 -- Comment les taches se font Workflow : l'agent propose, vous decidez.

  • 4.6 -- Base de donnees comme source de verite Schema PostgreSQL, scripts CLI, pourquoi DB > fichiers .md.

Partie C -- Workflows et routines

  • 4.7 -- Reconnaitre une routine Se repete 3+ fois, schema previsible = candidat a l'automatisation.

  • 4.8 -- Dry run avant confiance Tester en mode "montre-moi" avant de laisser l'agent agir seul.

  • 4.9 -- WORKFLOWS.md Documenter chaque procedure. Format standardise.

  • 4.10 -- Le rythme hebdomadaire Preview lundi, review vendredi. L'agent prepare les deux.

Partie D -- Securite et confiance

  • 4.11 -- La confiance est une configuration Pyramide des droits. Chaque niveau a ses regles.

  • 4.12 -- Le boundary prompt Ce que l'agent ne doit JAMAIS faire. Ecrit une fois, applique toujours.

  • 4.13 -- Audit : que peut acceder votre agent ? Prompt auto-audit. Verifier realite vs intention.

  • 4.14 -- Configuration bilingue Francais, anglais, ou les deux. Quand basculer.

Specifications locales : voir sections/07-localisation/
