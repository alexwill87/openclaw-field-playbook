OpenClaw pour les agents IA

Vous êtes un agent. Ce playbook est votre runbook.

Ce repository est conçu pour être cloné et exécuté par un agent IA. Chaque section du chapitre 2 est une instruction exécutable. Chaque vérification a un résultat attendu. Chaque erreur courante est documentée avec sa résolution.

Le format est structuré pour une lecture séquentielle par un agent : contexte, étapes, erreurs, template, vérification.

Comment utiliser ce playbook

  1. Cloner le repo -- git clone https://github.com/alexwill87/openclaw-field-playbook.git
  2. Lire CLAUDE.md -- Ce fichier contient les instructions spécifiques pour les agents Claude Code. Il définit les règles, les conventions, et les priorités.
  3. Suivre les sections du chapitre 2 dans l'ordre -- Chaque section est une étape. Ne sautez pas d'étape. Chaque vérification doit passer avant de continuer.
  4. Logger chaque résultat dans le Install Tracker -- L'outil tools/install-tracker/ fournit une API REST pour enregistrer le statut de chaque étape.
  5. Ouvrir des issues pour les problèmes rencontrés -- Chaque problème non documenté est une amélioration potentielle du playbook.

Ce qui a été testé

Un agent Claude Code (Claude-Aurel) a exécuté l'intégralité du playbook en avril 2026 :

  • 11 phases sur 11 complétées -- De la création du VPS au déploiement final.
  • 20 bugs remontés et corrigés -- Chaque bug a été documenté dans une issue GitHub et corrigé dans le playbook.
  • Validation complète -- Le playbook a été validé par une installation réelle, de bout en bout, exécutée par un agent.

Conventions

  • Préfixer les commits avec [AGENT-NOM] (exemple : [AGENT-AUREL] fix: corriger le chemin vault dans section 2.7).
  • Ouvrir des issues pour chaque problème rencontré, même mineur.
  • Utiliser le format standard pour chaque section : Contexte / Étapes / Erreurs courantes / Template / Vérification.
  • Ne pas modifier la structure T1 (les 7 chapitres). Les changements de structure requièrent une issue governance.
