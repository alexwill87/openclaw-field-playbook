OpenClaw pour les agents IA

Vous etes un agent. Ce playbook est votre runbook.

Ce repository est concu pour etre clone et execute par un agent IA. Chaque section du chapitre 2 est une instruction executable. Chaque verification a un resultat attendu. Chaque erreur courante est documentee avec sa resolution.

Le format est structure pour une lecture sequentielle par un agent : contexte, etapes, erreurs, template, verification.

Comment utiliser ce playbook

  1. Cloner le repo -- git clone https://github.com/alexwill87/openclaw-field-playbook.git
  2. Lire CLAUDE.md -- Ce fichier contient les instructions specifiques pour les agents Claude Code. Il definit les regles, les conventions, et les priorites.
  3. Suivre les sections du chapitre 2 dans l'ordre -- Chaque section est une etape. Ne sautez pas d'etape. Chaque verification doit passer avant de continuer.
  4. Logger chaque resultat dans le Install Tracker -- L'outil tools/install-tracker/ fournit une API REST pour enregistrer le statut de chaque etape.
  5. Ouvrir des issues pour les problemes rencontres -- Chaque probleme non documente est une amelioration potentielle du playbook.

Ce qui a ete teste

Un agent Claude Code (Claude-Aurel) a execute l'integralite du playbook en avril 2026 :

  • 11 phases sur 11 completees -- De la creation du VPS au deploiement final.
  • 20 bugs remontes et corriges -- Chaque bug a ete documente dans une issue GitHub et corrige dans le playbook.
  • Validation complete -- Le playbook a ete valide par une installation reelle, de bout en bout, executee par un agent.

Conventions

  • Prefixer les commits avec [AGENT-NOM] (exemple : [AGENT-AUREL] fix: corriger le chemin vault dans section 2.7).
  • Ouvrir des issues pour chaque probleme rencontre, meme mineur.
  • Utiliser le format standard pour chaque section : Contexte / Etapes / Erreurs courantes / Template / Verification.
  • Ne pas modifier la structure T1 (les 7 chapitres). Les changements de structure requierent une issue governance.
